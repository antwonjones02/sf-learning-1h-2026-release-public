Learning Permissions Delivered For Use In A Later Release

Feature #14 of 32
Admin

What Is This Feature?

Four permissions — Activate Multiple Employments, View User Employment, View Person Connector, and Edit Person Connector — have been included in system default roles over recent releases to support a future person-centric experience in Learning. No functionality is activated by these permissions in this release; they can be safely ignored. These permissions support the future adoption of person-centric identity management.

What Is Changing?

Four permissions are pre-delivered in system default roles: Activate Multiple Employments (since 1H 2025), View User Employment (since 2H 2024), View Person Connector (since 1H 2025), and Edit Person Connector (since 1H 2025). No action is required as the associated features are not yet delivered.

Why Should We Care?

Pre-delivering these permissions ensures system default roles are ready for upcoming person-centric Learning features when they are released, minimizing future configuration effort.

Risk of Inaction

No action needed — automatically enabled. These permissions can be safely ignored until the associated functionality is delivered in a future release.
