Support for Integer and Decimal Custom Fields for User Entity

Feature #5 of 32
Admin

What Is This Feature?

Admins can now create custom fields of type Integer and Decimal for the User entity, in addition to the existing String type. The new Data Type selector (String, Integer, Decimal) appears when creating or editing custom fields in System Administration > Custom Fields > Users. Integer and decimal custom fields are supported in user searches, assignment profile rules, and standard and custom reports.

What Is Changing?

A new Data Type field is added when creating or editing user custom fields. Previously, user custom fields were always String type. Decimal and integer types are now available and supported in searches, assignment rules, and reports.

Why Should We Care?

Organizations can now store numeric data (such as headcount codes or percentages) as proper numeric types, enabling more precise filtering, dynamic assignment profiles, and richer reporting without manual data conversion.
