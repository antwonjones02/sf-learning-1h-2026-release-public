Bulk Delete Unused iContent Files From Learning Administration

Feature #1 of 32
Admin

What Is This Feature?

Admins can now identify and bulk delete unused or irrelevant content files stored on iContent servers directly from the iContent Management page (System Administration > System Management Tools). Deleting iContent files removes the files along with any linked content objects and packages, inactivates the associated courses, and causes learners to lose in-progress course progress. Reactivation requires manual cleanup of the course structure. Based on Customer Influence Idea 267806.

What Is Changing?

A new iContent Management page in Learning Administration allows admins to view and bulk delete unused iContent files. Previously, there was no way to view or delete iContent files directly from Learning Administration.

Why Should We Care?

Organizations with large content libraries can reclaim storage space and reduce iContent server volume by removing stale or unused content, improving system performance and maintainability.

Risk of Inaction

Without enabling this feature, unused iContent files continue to consume storage. Admins should plan carefully before deleting, as it inactivates associated courses and affects learners with in-progress work.
