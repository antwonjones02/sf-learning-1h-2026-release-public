Bulk Delete Unused iContent Files From Learning Administration
Enable New Major Admin

What is this feature?

Admins can now identify and bulk delete unused or irrelevant content files stored on iContent servers directly from the iContent Management page (System Administration > System Management Tools). Deleting iContent files removes the files along with any linked content objects and packages, inactivates the associated courses, and causes learners to lose in-progress course progress. Reactivation requires manual cleanup of the course structure. Based on Customer Influence Idea 267806.

Why should we care?

Organizations with large content libraries can reclaim storage space and reduce iContent server volume by removing stale or unused content, improving system performance and maintainability.

Risk of Inaction

Without enabling this feature, unused iContent files continue to consume storage. Admins should plan carefully before deleting, as it inactivates associated courses and affects learners with in-progress work.

Auto-Purge Archived Connector Input Files
Enable Changed Minor Admin

What is this feature?

Archived connector input files are now automatically purged after 30 days by default, eliminating the need for manual retention configuration. When the Enable input file archive option is activated, the previously configurable Purge archived input files older than setting is removed. SAP recommends downloading the zip file of archived input files before the 30-day window closes for troubleshooting purposes.

Why should we care?

Automatic purging of archived connector files reduces manual admin overhead and keeps the system lean, preventing accumulation of stale files that can impact performance.

Risk of Inaction

Without enabling the archive feature, connector input files are not archived and therefore not subject to auto-purge. If archiving is in use, the fixed 30-day window replaces any previously configured retention period.

Connectors Now Process User Records With Future Termination Dates
Enable Changed Minor Admin

What is this feature?

The User Connector - SF and User Connector now process active user records (NOTACTIVE = N) even when TERM_DTE is set to a future date. Previously these records were rejected, requiring manual removal of the termination date. A warning message appears in the Connector Run Details report, and the termination date in Learning is only updated once the user becomes inactive. The field mapping sfuser.field.mapping.data.TERM_DTE must be mapped to Platform column TERM_DTE.

Why should we care?

Organizations can sync HR system records for employees on notice periods without manual workarounds, improving data accuracy and reducing connector maintenance effort.

Risk of Inaction

Without enabling this feature, user records with future termination dates will continue to be rejected during connector runs, requiring manual intervention to process them.

Deprecation of SuccessFactors Learning Token Server
Migrate to OAuth NOW Changed Minor Admin

What is this feature?

The SuccessFactors Learning token server is deprecated and will be deleted on November 20, 2026. The Learning token server uses HTTP basic authentication (Client ID and Secret), which is less secure than the platform token server's support for SAML Assertion Based token requests. Customers should migrate integrations to OAuth 2.0 or OpenID Connect authentication using the platform token server.

Why should we care?

The platform token server provides more secure authentication mechanisms (SAML Assertion Based) compared to the legacy Learning token server, reducing security risk for API integrations.

Risk of Inaction

Integrations using the Learning token server for authentication will stop working after November 20, 2026. Organizations must plan and execute migration to OAuth 2.0 or OpenID Connect before the deletion date.

Testing Strategy

SAP provides a 4–5 week preview window before each production release. For 2H 2025, the preview environment is available October 17, 2025, with production going live November 14–15, 2025, giving your team a 28-calendar-day (20 business-day) testing window. Use this time to validate new features, regression-test existing configurations, and prepare communications.

1

Pre-Release Preparation

Before October 17
  • Review all features on this site and identify those relevant to your organization
  • Prioritize features by filtering on Action Required, these need decisions before go-live
  • Ensure your preview (test) environment is aligned with production, SAP recommends refreshing preview at least once annually
  • Identify test owners: system admins, learning admins, integration teams, and business process owners
  • Document your current configurations and key business processes for regression comparison
2

Preview Environment Testing

October 17 – November 13
  • Validate Universal features, these are automatically enabled; confirm they work as expected with your configuration
  • Evaluate Opt-In features, enable in preview, test, and decide whether to activate for production
  • Regression-test core workflows, SAP recommends testing end-to-end business processes, not just isolated features (Effective People)
  • Test integrations, verify all API connections, especially if using the Learning Token Server (migration to OAuth 2.0 is required before November 20, 2026)
  • Validate roles and permissions, confirm role-based permissions are correctly configured for any new features
  • Test across experiences: verify End User, Admin, and Manager views using the Experience filter on this site
3

Stakeholder Review & Sign-Off

November 1 – November 13
  • Share test results with business process owners for validation
  • Document decisions on Opt-In features: enable, defer, or decline
  • Coordinate with HR, IT, security, and payroll teams on cross-module impacts (SAP Community)
  • Prepare internal communications and training materials for end users
4

Post-Production Validation

November 14–15 onward
  • Verify production deployment, SAP delivers releases regionally over the weekend to minimize disruption (Zalaris)
  • Run smoke tests on critical workflows immediately after go-live
  • Monitor for release-introduced issues, note that some bugs may not be addressed until the 1H 2026 release
  • Submit SAP support cases promptly for any issues found in production
!

Token Server Migration

Complete before Nov 20, 2026
  • The SAP SuccessFactors Learning Token Server will be permanently deleted on November 20, 2026
  • All integrations using legacy token-based authentication must migrate to OAuth 2.0
  • Begin migration planning immediately, test OAuth 2.0 connections in preview before production cutover
  • Coordinate with integration vendors and internal development teams
