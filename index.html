Frequently Asked Questions

Required features are enabled automatically for all customers. No admin action is required, they will appear in your system after the release goes live.

Optional features are available but must be manually enabled by a system administrator. Review each feature and decide whether to enable it for your organization.

Premium features require additional licensing or are restricted to specific product editions. Contact your SAP account representative to discuss availability and pricing.

Deprecation means a feature is being removed or has reached end of life. Migration or alternative action is typically required before the removal deadline.

Preview environments receive the update on April 13, 2026. Production systems go live May 15–16, 2026. The Token Server hard-delete deadline is November 20, 2026.

The SAP SuccessFactors Learning Token Server will be permanently deleted on November 20, 2026. Any integrations still using legacy token-based authentication will break. All organizations must complete migration to OAuth 2.0 before this date. This is the highest-priority action item in this release.

Testing Strategy

SAP provides a 4–5 week preview window before each production release. For 1H 2026, the preview environment is available April 13, 2026, with production going live May 15–16, 2026, giving your team a 32-calendar-day (25 business-day) testing window. Use this time to validate new features, regression-test existing configurations, and prepare communications.

1

Pre-Release Preparation

Before April 13
  • Review all features on this site and identify those relevant to your organization
  • Prioritize features by filtering on Action Required, these need decisions before go-live
  • Ensure your preview (test) environment is aligned with production, SAP recommends refreshing preview at least once annually
  • Identify test owners: system admins, learning admins, integration teams, and business process owners
  • Document your current configurations and key business processes for regression comparison
2

Preview Environment Testing

April 13 – May 14
  • Validate Required features, these are automatically enabled; confirm they work as expected with your configuration
  • Evaluate Optional features, enable in preview, test, and decide whether to activate for production
  • Regression-test core workflows, SAP recommends testing end-to-end business processes, not just isolated features
  • Test integrations, verify all API connections, especially if using the Learning Token Server (migration to OAuth 2.0 is required before November 20, 2026)
  • Validate roles and permissions, confirm role-based permissions are correctly configured for any new features
  • Test across experiences: verify End User, Admin, and Manager views using the Experience filter on this site
3

Stakeholder Review & Sign-Off

May 1 – May 14
  • Share test results with business process owners for validation
  • Document decisions on Optional features: enable, defer, or decline
  • Coordinate with HR, IT, security, and payroll teams on cross-module impacts (SAP Community)
  • Prepare internal communications and training materials for end users
4

Post-Production Validation

May 15–16 onward
  • Verify production deployment, SAP delivers releases regionally over the weekend to minimize disruption
  • Run smoke tests on critical workflows immediately after go-live
  • Monitor for release-introduced issues, note that some bugs may not be addressed until the 2H 2026 release
  • Submit SAP support cases promptly for any issues found in production
!

Token Server Migration

Complete before Nov 20, 2026
  • The SAP SuccessFactors Learning Token Server will be permanently deleted on November 20, 2026
  • All integrations using legacy token-based authentication must migrate to OAuth 2.0
  • Begin migration planning immediately, test OAuth 2.0 connections in preview before production cutover
  • Coordinate with integration vendors and internal development teams
